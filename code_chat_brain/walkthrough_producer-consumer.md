# Asynchronous Threading Implementation

We have successfully overhauled `speech_synthesis.py` with the highly optimized Producer-Consumer threading pattern!

## Architectural Changes

- **Background Audio Extractor**
  We created `audio_extractor_worker()`, which runs on a background daemon thread. It loops through the `segments` list and runs the FFmpeg extraction subroutine seamlessly.
  
- **Queue System**
  The background worker feeds the completed `.wav` paths and metadata into a `queue.Queue(maxsize=5)`. The queue acts as a strict buffer—it allows the CPU to pre-fetch up to 5 segments ahead of the GPU. If the GPU falls behind, the CPU pauses automatically, ensuring we don't spam your hard drive with hundreds of temporary audio files at once!

- **Synchronous Timeline**
  The main thread acts as the Consumer. It continuously polls the queue, triggering the heavy `cosyvoice.inference_zero_shot()` process. Since it receives segments sequentially, the exact absolute timeline padding we implemented previously works flawlessly with the exact duration generated by the GPU.

## Verification Required

> [!IMPORTANT]
> The background extraction thread is fully detached and handles its own exception wrapping. If an FFmpeg extraction fails, it safely passes the error downstream through the queue so the main thread can log it and cleanly skip the segment without crashing the main loop.
